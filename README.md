# Claude Configs

一套围绕 Claude Code 搭建的个人工作系统。不只是配置文件，更是一个**可进化的 AI 协作方法论**。

## 核心理念

与 AI 对话是一次性的，但好的工作流应该可复用、可迭代。这个仓库沉淀了我与 Claude 协作中积累的原则、流程和规范。

**三件事：**

- **写文档** — 把模糊的协作直觉写成可执行的规范（执行规范、环境配置、隐私策略）
- **沉流程** — 跑通一次 → 记录成流程 → 下次直接复用 → 持续迭代
- **记重点** — 每次收束对话，把做了什么记下来，按月归档，可追溯

## 有什么

### 配置文档

| 文件 | 作用 |
|------|------|
| `CLAUDE.md` | 全局配置入口，快速导航 |
| `sop.md` | 通用执行规范 |
| `env.md` | 环境配置（Python/OCR/API） |
| `privacy.md` | 隐私规范（第三方上传限制） |
| `anti-corruption.md` | 对话防腐化机制 |
| `checkpoint.md` | 会话状态检查点 |
| `settings.json` | Claude Code 设置 |

### 流程沉淀系统

`flow-records/` 下记录了跑通的工作流，分两类：

- **通用流程** — 跨项目可复用的 SOP（如任务执行规范、流程沉淀方法）
- **项目特有** — 特定场景的解决方案（如 MinerU 合同 PDF 批量 OCR）

每条记录可以进化：draft → stable → deprecated，可从项目级升级为通用级。

### 会话摘要

`session-highlights/` 按月度存档每次收束对话的关键改动，让自己能回头看。

### 技能

skills/ 下安装了 Claude 官方及社区的技能插件，扩展 Claude 的能力边界（头脑风暴、PDF 处理、PPT 生成、Excel 操作等）。

## 工作流

```
跑通 → 沉淀 → 复用 → 进化
```
