# Claude Configs

> 把一次性的 AI 对话，沉淀成可复用、可迭代、可迁移的个人工作系统。

这不是一份简单的 Claude Code 配置备份，而是一套围绕 **SOP、隐私边界、流程沉淀、会话记忆、技能扩展** 搭建的 AI 协作方法论。它适合所有希望把 Claude Code / AI Agent 从“临时帮手”升级成“长期工作伙伴”的人。

如果你也在反复遇到这些问题：

- 每次开新会话都要重新解释背景、偏好和边界；
- 跑通过的任务，下次仍然要从零摸索；
- AI 很容易越界：上传敏感资料、忽略检查点、忘记验证；
- 配置、技能、流程散落在不同地方，无法系统复用；

这个仓库可以作为一个可参考、可 fork、可改造的模板。觉得有启发的话，欢迎 ⭐ Star，后续会持续沉淀更多真实工作流。

## 亮点

- **开箱即用的协作入口**：`CLAUDE.md` 作为全局导航，让 Agent 快速理解规则、路径和工作方式。
- **可执行 SOP**：把“先确认、再执行、最后验证”的经验写成明确流程，降低遗漏和误操作。
- **隐私优先**：针对合同、证件、财报、OCR 上传等高风险场景，明确第三方 API 上传边界。
- **流程沉淀系统**：任务跑通后沉淀为 `flow-records/`，下次不再重复踩坑。
- **会话摘要机制**：用 `session-highlights/` 记录关键变更，解决长周期工作中的上下文遗忘。
- **技能化扩展**：通过 `skills/` 管理 PDF、Excel、PPT、调试、计划执行等可复用能力。

## 适合谁

- 正在使用 Claude Code、Codex、Gemini CLI 等 Agent 工具的人；
- 希望搭建个人 AI 工作流，而不只是保存几条 prompt 的人；
- 需要处理 PDF、Excel、OCR、文档归档、流程化交付的人；
- 关注隐私边界，不希望 AI 默认把敏感材料上传到第三方服务的人；
- 想把“这次做成了”变成“以后可以稳定复用”的人。

## 仓库结构

| 路径 | 作用 |
|------|------|
| `CLAUDE.md` | 全局配置入口，负责快速导航到各类规范 |
| `sop.md` | 通用执行规范，定义任务分流、检查点和验证方式 |
| `privacy.md` | 隐私规范，约束第三方 API 上传和敏感材料处理 |
| `env.md` | 环境约定，记录本地工具链、PDF/OCR/API 使用偏好 |
| `anti-corruption.md` | 对话防腐化机制，减少长会话跑偏 |
| `checkpoint.md` | 会话状态检查点，用于阶段性同步与恢复 |
| `flow-records/` | 可复用流程库，记录已经跑通的工作流 |
| `session-highlights/` | 会话摘要，按月记录关键产出和变更 |
| `skills/` | Claude/Agent 技能扩展，如 PDF、Excel、PPT、调试等 |
| `settings.json` | Claude Code 相关设置 |

## 核心方法论

### 1. 写文档：把直觉变成规则

很多 AI 协作问题并不是模型能力不足，而是规则没有写清楚。本仓库把常见偏好、边界和执行方式文档化，让 Agent 每次进入任务时都有统一上下文。

### 2. 沉流程：把一次成功变成可复用资产

一次任务跑通后，不只保留结果，还会记录触发条件、执行步骤、脚本参数、踩坑笔记和验证方法。这样下一次遇到类似任务时，可以直接复用流程，而不是重新试错。

### 3. 记重点：让长期协作可追溯

会话结束时记录关键改动、产物和决策。长期看，这些摘要会形成一条可回放的工作轨迹，帮助快速恢复上下文。

## 工作流

```text
明确边界 → 执行任务 → 验证结果 → 沉淀流程 → 下次复用
```

更简洁地说：

```text
跑通 → 沉淀 → 复用 → 进化
```

## 快速开始

1. Fork 或下载本仓库。
2. 从 `CLAUDE.md` 开始阅读，了解全局入口如何组织。
3. 根据自己的工具链修改 `env.md`、`sop.md` 和 `privacy.md`。
4. 把你跑通的任务记录到 `flow-records/`。
5. 每次重要会话结束后，在 `session-highlights/` 追加摘要。

> 建议：不要直接照搬所有内容。最好的方式是把它当成脚手架，保留结构，替换成你自己的工作习惯和隐私边界。

## 隐私与安全

本仓库已经尽量避免记录真实客户名、个人身份信息、绝对路径、密钥和完整业务数据。你在 fork 后也建议遵循同样原则：

- 不提交 `.env`、token、cookie、API key；
- 不提交完整合同、证件、财报、客户清单等原始资料；
- 流程示例使用脱敏名称和占位路径；
- 涉及第三方 OCR/API 上传时，优先限制页码范围；
- 定期用 `rg` 或 secret scanning 工具检查仓库。
