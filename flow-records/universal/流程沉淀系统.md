## 摘要

把跑通的工作流沉淀为可复用的记录，自动分流到通用（→ 全局 CLAUDE.md + skill）或项目特有（→ 项目 CLAUDE.md）。

## 触发条件

每次跑通一个流程后，你想把它记下来供以后复用。

## 输入

- 刚跑通的工作流上下文（由我说或你输入）
- 终端运行 `!precipitate` 或跟我说"沉淀这个流程"

## 执行步骤

### 沉淀（创建）

1. **我主动问「要不要沉淀」**（可选，流程跑通后我会提）
2. **你触发** → 跟我说"沉淀这个流程"（我补上下文）或终端 `!precipitate`（交互式问答）
3. **我/脚本判断类型**：
   - `universal` → 跨项目通用，入 `flow-records/universal/` + 全局 CLAUDE.md + 生成 skill 骨架
   - `project-specific` → 仅当前项目，入 `flow-records/project-specific/` + 项目 CLAUDE.md
4. **你或我**编辑记录文件补全步骤细节和踩坑笔记

### 进化（更新）

1. **跟我说"进化这个流程"** → 我读现有记录，根据对话上下文补充变化
2. **终端运行 `!evolve`** → 交互式选择操作:
   - **标记状态**: draft → stable → deprecated
   - **类型升级**: project-specific → universal（同时创建 skill）
   - **加版本日志**: 自动更新 version + 写 changelog
   - **查看/编辑**: 直接读或 vim 编辑记录文件

## 预期产出

- 记录文件在 `~/.claude/flow-records/` 对应目录下
- 索引自动更新到 `_index.md`
- 全局/项目 CLAUDE.md 追加入口

## 验证方法

- 打开 `~/.claude/flow-records/_index.md` 看有没有新条目
- 通用流程的话检查 `~/.claude/CLAUDE.md` 有没有追加
- 项目特有检查对应项目目录的 CLAUDE.md

## 踩坑笔记

- 脚本的 sed 占位符必须与 _index.md 完全一致，改 _index.md 时记得同步脚本
- Git Bash 下 sed 的 `\\n` 换行可能变成字面字符串，涉及 YAML 编辑时用 awk 替代
- 记录文件创建后我会帮忙补充步骤细节，但最终填充还得靠你的实际操作经验
- skill 骨架只生成不注册，需要手动 `npx skills` 注册（可以自动化，但暂未做）
- 脚本依赖 bash + sed + awk，Windows 下需 Git Bash 环境
- `evolve` 的类型升级操作不会自动删除旧索引条目，需确认后手动清理
