# 用户隐私规范

> 涉及第三方 API 上传（MinerU OCR等）时的隐私保护守则。

## 核心原则

**绝不将完整敏感文档上传至第三方服务。** 合同、身份证、财报等含敏感信息的文件，上传前必须限制页码范围。

## MinerU OCR 上传规范

### 上传前审查

1. **查页数** — 先看 PDF 总页数（PyMuPDF 或 `pdfinfo`）
2. **评估敏感度** — 合同含：公司名、法人、金额、地址、银行账号、联系方式等
3. **判断需要范围** — 通常只传能提取标题/摘要的页数即可

### 上传策略

| 场景 | 推荐页数 | 说明 |
|------|---------|------|
| 提取标题 | 第1页 | 标题通常在第一页 |
| 提取签章信息 | 最后一页 | 签章页通常在末页 |
| 提取关键条款 | 第1页 + 签章页 | 必要时加1-2页关键页 |
| 全文提取 | **禁止默认操作** | 除非用户明确要求并经确认 |

### API 参数示例

```json
{
  "file_name": "合同.pdf",
  "language": "ch",
  "page_range": "1",      // ← 只传第1页
  "enable_table": false,
  "is_ocr": true,
  "enable_formula": false
}
```

### 不得上传的内容

- 含身份证号、银行账号、护照号的页面
- 含个人签名的页面（除非明确需要核验）
- 含完整财务数据的多页报表
- 含商业秘密的技术细节页

### 违例处理

如果发现有代码/流程尝试上传超过用户常规范围的页数：
1. 拦截操作
2. 提醒用户当前页码范围
3. 等待用户确认后再执行

## 适用范围

- MinerU API OCR
- 其他任何将文件内容发送至第三方 API 的操作
- 本地处理的 PDF（PyMuPDF / pdftotext）不受此限制
